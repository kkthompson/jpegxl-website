# Cutover parity audit — legacy root site vs Astro build

*Audited 2026-07-07 against `site/dist` built from the current working tree.*

Question answered: **if GitHub Pages switched from serving the repo root to serving the Astro build today, what would break?** Answer: nothing user-facing, after the three gap fixes noted below (now applied).

## Content drift

Visible text of every migrated page was diffed against its root twin:

- **14 of 17 pages: zero drift** (identical visible wording).
- `art/index.html`: the Astro version has *newer* copy (gallery intro, WebAssembly-decoder note). The root page is the stale one. No root-side content is lost.
- `battle-of-codecs.html`, `hdr-test-page.html`: the only root-side extras are dead Webflow template footer links ("Cookies Settings", "Terms of Service", "Privacy Policy" — all `href="#"`). Intentionally dropped by the shared Astro footer.
- The April 2026 "Removing Webflow dependecies" commit to root pages was verified to be purely mechanical (attribute/script removal) — it introduced no content the Astro pages lack.

## Bottom line

The Astro build is a superset of the live site's real content. Remaining cutover steps:

1. Commit the working tree (includes untracked-but-required source: `src/pages/art/`, `src/content/`, `content.config.ts`, `public/robots.txt`, `public/images/art/`).
2. Enable the CI workflow (`.github/workflows/ci.yml`, already written) and add a deploy workflow that publishes `site/dist` to GitHub Pages.
3. After cutover is verified: delete the legacy root site (including `phoenix/` and the root asset copies) and update the sitemap URL in Search Console.
